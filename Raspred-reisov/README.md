# Логистический помощник — Система распределения рейсов

## Запуск

Открыть **два терминала**.

### Терминал 1 — Бэкенд (Node.js)
```bash
cd backend-node
node server.js
```
Сервер стартует на http://localhost:8000, создаёт БД и заполняет тестовыми данными (8 машин, 12 маршрутов).

### Терминал 2 — Фронтенд
```bash
cd frontend
npm install
npm run dev
```
Открыть в браузере: **http://localhost:5173**

---

## Что делает система

1. **Карта** — все машины на карте Москвы с цветовой кодировкой:
   - 🟢 Свободна
   - 🔵 В рейсе
   - 🟠 ТО
   - 🔴 Запрет выпуска (механик)

2. **Таблица маршрутов** — список всех рейсов с предложенными машинами, score-баром и кнопками подтверждения/отклонения.

3. **Кнопка «Пересчитать»** — запускает алгоритм scoring, система предлагает оптимальные назначения (основное + 2 альтернативы).

4. **Блокировка** — при подтверждении рейса ставится soft lock на 3 минуты, второй логист видит «Занято».

5. **WebSocket** — все изменения в реальном времени у всех подключённых пользователей.

## Алгоритм scoring

```
Score = 0.40 × DistanceScore   (меньше пробег = выше)
      + 0.25 × ProfitScore     (выше прибыль = выше)
      + 0.20 × RestrictionScore (меньше ограничений = выше)
      + 0.10 × PriorityScore   (приоритет 1-5)
      + 0.05 × StabilityScore  (стабильность машины)
```

Жёсткие фильтры (hard filters): статус машины, разрешение механика, грузоподъёмность, аккредитации.
